# Atlas migrations

Versioned, reviewable migrations generated by [Atlas](https://atlasgo.io/) from the single source of
truth `../schema.sql`, applied with `atlas migrate apply` and gated in CI by `atlas migrate lint`.

**Phase 0:** this directory is intentionally empty. `pixela migrate` bootstraps the *initial* schema by
applying the embedded `db/schema.sql` transactionally (idempotent), so the binary needs no `atlas`
runtime dependency (single `CGO_ENABLED=0` distroless image). See
[`docs/architecture/go-backend.md` §8.4](../../../../docs/architecture/go-backend.md) (Phase-0 deviation note).

**From the first schema change (Phase 1+):** generate a versioned migration and commit it here:

```bash
atlas migrate diff <name> \
  --to "file://db/schema.sql" \
  --dev-url "docker://postgres/16/dev" \
  --dir "file://db/migrations"
```
