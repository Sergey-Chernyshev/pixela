# Agent 06 — Frontend Shell

**Фаза:** 4 · **Фичи:** F-20..F-23, F-35, F-36, F-44 · **Зависит от:** Agent 02 (API для чтения)

## Задача
Каркас Angular-дашборда: аутентификация, список проектов, лента билдов с фильтрами, детали билда со
списком снапшотов. Подготовка к review UI (Agent 07).

## Контекст (прочитать)
- `specs/08-frontend-spec.md` (экраны 1-2), `frontend/angular-decisions.md`, `specs/04-api-contract.md`
  (endpoints дашборда), `skills/how-to-use-your-skills.md` (применить frontend-design skill).

## Что сделать
1. **Типизированный API-клиент** в `core/api/` — методы под endpoints дашборда; типы в `core/models/`
   зеркалят `04-api-contract.md`.
2. **Auth** (F-35): login-экран (email+пароль), auth-сервис на signals, HTTP-interceptor (токен/cookie),
   route guard. GitLab OAuth-кнопка — заглушка под F-43 (полноценно в Фазе 5/6).
3. **Список проектов** (F-20): только проекты пользователя.
4. **Лента билдов** (F-21): таблица/карточки, фильтры (branch, status), поиск по commit, server-side
   пагинация, статус-бейджи цветом+иконкой+текстом, ссылка на CI-джобу.
5. **Детали билда** (F-22, F-23): счётчики, тоггл «только изменившиеся» (default ON), сетка превью снапшотов
   с ленивой загрузкой и виртуализацией (CDK virtual-scroll), клик → review (роут под Agent 07).
6. **Live-обновление** (F-44): polling статуса COMPARING (SSE — улучшение, B-11).
7. Signals + OnPush везде. Без NgModules. Только CDK, без тяжёлых китов.

## Acceptance criteria
- [ ] Вход по email+паролю; неавторизованный редиректится на login; interceptor добавляет токен.
- [ ] Пользователь видит ТОЛЬКО свои проекты (сервер фильтрует, фронт не показывает чужое).
- [ ] Лента билдов фильтруется по branch/status, пагинируется server-side.
- [ ] Детали билда показывают счётчики и список; тоггл «только изменившиеся» работает.
- [ ] Сетка превью не тормозит на сотнях снапшотов (lazy + виртуализация).
- [ ] Статус COMPARING обновляется без перезагрузки страницы (polling).
- [ ] Типы фронта соответствуют API-контракту; всё на signals + OnPush.
- [ ] Применён frontend-design skill — UI не «генерик».

- Не тащить NgRx/Angular Material целиком.
