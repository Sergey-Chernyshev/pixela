# 11 — Non-Functional Requirements

## Производительность и масштаб (целевые, не гипермасштаб)

- Билд до **~500 скриншотов** обрабатывается комфортно (минуты, не часы).
- До **~50 активных проектов**, история на месяцы.
- Дашборд: список билдов и детали открываются < 1 с при типичных объёмах.
- Review UI: переключение между изменившимися снапшотами плавное (благодаря prefetch + thumbnails).

Достигается за счёт:
- async diff в очереди + горизонтально масштабируемые воркеры;
- CAS-дедупликация (неизменившиеся блобы не льются и не дублируются);
- thumbnails для сетки превью (не полноразмерные PNG);
- пагинация и виртуализация списков;
- индексы БД на частых запросах (см. `03-data-model.md`).

## Надёжность

- Падение одного diff-job не валит билд (изоляция ошибок, ретраи).
- Падение уведомления не ломает обработку билда.
- Идемпотентность ingestion: ретрай CI-джобы безопасен (upsert).
- При рестарте сервисов незавершённые jobs переподхватываются из очереди (BullMQ persistence в Redis).
- Healthchecks на api/worker/postgres/redis/minio в docker-compose.

## Хранилище и его рост

- Главный потребитель места — PNG в MinIO. CAS снижает дублирование, но история растёт.
- Стратегия ретенции (пост-MVP, но заложить): чистка diff-PNG старых билдов; baseline не трогать.
- Мониторить размер бакета; документировать в `infra/deployment.md`.

## Бэкап и восстановление (F-40)

- **Postgres**: регулярный `pg_dump` (cron/скрипт), хранить вне сервера.
- **MinIO**: бэкап бакета (mc mirror / реплика) — без блобов история diff/baseline теряется.
- Документировать процедуру восстановления: поднять compose → восстановить БД → восстановить бакет.
- Тест восстановления хотя бы раз (иначе бэкап = надежда, а не гарантия).

## Наблюдаемость

- Структурированные логи (JSON) с уровнями. Без секретов.
- Метрики (пост-MVP): длина очереди, время diff-job, число билдов по статусам.
- Эндпоинт `/health` (api и worker).

## Совместимость и версии

- Node LTS (зафиксировать в `.nvmrc` / `engines`).
- Зафиксированные версии pixelmatch/sharp/libvips ради детерминизма diff (см. `07-diff-engine.md`).
- Playwright reporter совместим с актуальной мажорной версией Playwright; задокументировать поддерживаемый диапазон.

## Доступность (a11y) дашборда

- Хоткеи review UI — с видимой подсказкой.
- Контраст статус-бейджей достаточный; не полагаться только на цвет (иконка + текст).
- Angular CDK a11y (focus trap в модалках, aria на интерактивных элементах).
