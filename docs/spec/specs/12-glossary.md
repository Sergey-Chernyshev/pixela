# 12 — Glossary

Единый словарь терминов. Используй ровно эти слова в коде, API и UI (консистентность важна).

- **Project** — единица изоляции. Один тестируемый репозиторий/приложение. Имеет API-ключи и участников.
- **Build** — один прогон визуальных тестов (обычно = один CI-пайплайн). Содержит набор снапшотов. Имеет ветку, коммит, ciBuildId, агрегатный статус.
- **Snapshot** — один скриншот в рамках билда: уникален по (build, name, browser, viewport). Имеет статус (unchanged/changed/new/...).
- **Baseline** — принятый (approved) эталонный скриншот для (project, branch, name, browser, viewport). То, с чем сравнивается новый снапшот.
- **Image** — content-addressable блоб PNG, хранится по sha256. На один и тот же байт-контент — один Image (дедупликация). Снапшоты и baseline ссылаются на Image.
- **Diff / diff image** — изображение различий, генерируется при CHANGED. Подсвечивает изменившиеся пиксели.
- **diffRatio** — доля изменившихся пикселей [0..1]. `diffPixels / (width*height)`.
- **diffPixels** — абсолютное число пикселей, признанных изменившимися (по pixelThreshold).
- **pixelThreshold** — порог pixelmatch: насколько должен отличаться ОДИН пиксель, чтобы считаться изменённым (борьба с anti-aliasing-шумом).
- **diffRatioThreshold** — какая доля изменённых пикселей допустима, чтобы снапшот всё ещё считался UNCHANGED.
- **Approve / Reject** — решение ревьюера по изменившемуся снапшоту. Approve обновляет Baseline + готовит git-коммит.
- **ApprovalEvent** — запись аудита: кто/когда/какое решение принял по снапшоту.
- **Git-native baseline** — модель, где канонический baseline = файлы в git тестируемого проекта; сравнение всегда в пределах ветки; merge/rebase обрабатывает git. Противоположность серверному merge-base resolution.
- **CAS (content-addressable storage)** — хранение блоба по хэшу его содержимого; идентичные блобы не дублируются.
- **Ingestion** — приём скриншотов и метаданных из CI в Pixela (stateless, кладёт в очередь).
- **Worker** — процесс, выполняющий diff-jobs из очереди (BullMQ).
- **Shard** — часть параллельного прогона Playwright (`--shard=i/n`); несколько шардов агрегируются в один билд.
- **Soft mode** — режим, в котором визуальное различие не валит CI-прогон; ревью происходит в дашборде.
- **Onion-skin** — режим сравнения наложением с регулируемой прозрачностью.
- **Slider / шторка** — режим сравнения с перетаскиваемой вертикальной границей между baseline и new.
- **Ignore-область (маска)** — прямоугольник, исключаемый из сравнения (живые таймстемпы, рандомные id).
- **REMOVED** — статус: baseline для ключа есть, а снапшота с таким ключом в билде нет (пропал тест/скриншот).
