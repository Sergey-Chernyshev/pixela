# 07 — Diff Engine

> Diff — самая ДЕШЁВАЯ часть. Не переусложняй. Цель — корректность и детерминизм, не изобретательность.

## Движки

| Движок | Роль | Когда | Пакет |
|--------|------|-------|-------|
| **pixelmatch** | default | попиксельное сравнение с порогом | `pixelmatch` |
| **sharp** | обработка | декод PNG, resize, нормализация, кодирование diff | `sharp` |
| **odiff** | опциональный | layout/anti-aliasing aware, быстрый на больших картинках | `odiff-bin` |

MVP: только pixelmatch + sharp. odiff — за фиче-флагом per-project (F-46), не блокирует релиз.

## Алгоритм сравнения (worker job)

```
вход: snapshotId
1. загрузить Snapshot + найти Baseline по (project, branch, name, browser, viewport)
2. если baseline отсутствует → status = NEW, выход (diff не нужен)
3. скачать baseline.png и new.png из S3
4. декодировать обе через sharp → raw RGBA буферы
5. ПРОВЕРКА РАЗМЕРОВ:
   - если размеры различаются → это не «diff пикселей», это структурное изменение.
     Варианты: (a) пометить CHANGED с diffRatio=1.0 и сгенерировать side-by-side;
               (b) привести к общему холсту (padding), затем pixelmatch.
     Решение MVP: (a) — различие размеров = CHANGED, не пытаться «умно» ресайзить.
6. применить ignore-области (маски), если заданы: занулить пиксели в этих прямоугольниках в обоих буферах
7. pixelmatch(baselineBuf, newBuf, diffBuf, w, h, { threshold })
   → diffPixels (число несовпавших пикселей)
8. diffRatio = diffPixels / (w*h)
9. если diffRatio <= antialiasTolerance (или == 0) → status = UNCHANGED
   иначе → закодировать diffBuf в PNG (sharp), посчитать sha256, залить в S3,
           status = CHANGED, записать diffRatio, diffPixels, diffImageSha
10. обновить агрегатный статус билда
```

## Пороги (thresholds)

Два независимых параметра, оба конфигурируемы (project default + snapshot override):

- **`pixelThreshold`** (0..1, дефолт ~0.1) — порог pixelmatch: насколько отличается цвет одного пикселя,
  чтобы считать его «изменённым». Борется с anti-aliasing-шумом на уровне пикселя.
- **`diffRatioThreshold`** (0..1, дефолт 0) — какая доля изменённых пикселей допустима, чтобы всё ещё
  считать UNCHANGED. 0 = любой изменённый пиксель → CHANGED. Иногда ставят крошечное значение для шума.

> Не путать. `pixelThreshold` — про «насколько отличается пиксель». `diffRatioThreshold` — про «сколько
> пикселей отличается». Оба нужны.

## Детерминизм (критично для тестового инструмента)

Diff обязан быть **воспроизводим**: одни и те же два PNG → один и тот же результат всегда. Иначе
инструмент «флачит» и команда перестаёт ему верить. Требования:

- Фиксированная версия pixelmatch/sharp (lock-файл, не `^`).
- Никакой зависимости от платформы в декодировании (sharp детерминирован, но зафиксируй версию libvips через образ).
- `includeAA: true/false` в pixelmatch — выбрать ОДНО значение и не менять между прогонами.
- diff-PNG кодируется с фиксированными параметрами sharp (без рандомизации компрессии, влияющей на sha).

> Важно: детерминизм diff на сервере НЕ заменяет детерминизм СНЯТИЯ скриншота — это забота Playwright
> (frozen clock, отключённые анимации, фикс. шрифты/локаль). См. `playwright/fixtures-and-determinism.md`.

## Ignore-области (F-45)

Прямоугольники `{x, y, width, height}`, которые исключаются из сравнения (живые таймстемпы, рандомные id).

- Хранятся на уровне `Baseline` или per-snapshot правила проекта (по `name`).
- Применяются ДО pixelmatch: зануление/заливка одним цветом в обоих буферах.
- Задаются в дашборде поверх изображения (рисование рамки) → сохраняются → применяются к будущим diff.
- MVP может принимать их через конфиг/SDK; UI-рисование — Should, не Must.

## Производительность

- pixelmatch — CPU-bound. Воркеры масштабируются горизонтально (несколько `pixela-worker` контейнеров).
- BullMQ concurrency на воркер — конфигурируема (дефолт = число CPU).
- Большие картинки (>4 МП) — кандидаты на odiff (быстрее) или downscale-перед-diff (с осторожностью: влияет на чувствительность).

## Ошибки и изоляция

- Битый PNG / ошибка декода → `Snapshot.status = ERROR`, `errorMsg`, билд НЕ падает целиком.
- Job ретраится N раз (BullMQ backoff), затем ERROR.
- Один ERROR-снапшот переводит билд в REVIEW_REQUIRED (не PASSED), но не в ERROR (ERROR билда — только если систематический сбой).
